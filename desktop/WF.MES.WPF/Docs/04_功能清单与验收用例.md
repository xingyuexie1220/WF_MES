## 2. UAT 测试用例

以下用例建议在测试环境执行，结果打勾签字。

### 2.1 登录与会话

| # | 步骤 | 预期结果 | 通过 |
|---|------|----------|------|
| L1 | 正确账号密码登录 | 进入主界面，显示有权限菜单 | ☐ |
| L2 | 错误密码登录 | 提示失败，不写库会话 | ☐ |
| L3 | operator 首次登录 | 强制修改密码 | ☐ |
| L4 | 同账号在 A 机登录后再在 B 机登录 | B 机提示冲突；确认后 A 机会话失效 | ☐ |
| L5 | 重复双击 exe | 提示已在运行 | ☐ |
| L6 | 停止心跳/超时等待 | 提示会话失效并退出 | ☐ |

### 2.2 权限

| # | 步骤 | 预期结果 | 通过 |
|---|------|----------|------|
| P1 | 无条码菜单权限的用户登录 | 侧栏无条码模组 | ☐ |
| P2 | 有菜单无「保存附件」权限 | 无法上传 QA 资料 | ☐ |
| P3 | 有菜单无「审核」权限 | 无法 QA 确认/驳回 | ☐ |
| P4 | admin 保存角色权限 | 成功；非授权用户不能保存 | ☐ |

### 2.3 条码主流程

| # | 步骤 | 预期结果 | 通过 |
|---|------|----------|------|
| B1 | 新增客户并保存 | 列表可见 | ☐ |
| B2 | 新增料号规则（含流水号段） | 保存成功，QA 状态为待上传 | ☐ |
| B3 | 上传图纸 + 实物图 | 状态变为待 QA 审核 | ☐ |
| B4 | QA 确认通过 | 状态为已确认 | ☐ |
| B5 | 未确认料号尝试打印 | 提示待 QA 确认 | ☐ |
| B6 | 已确认料号生成条码 | 生成单与明细写入数据库 | ☐ |
| B7 | 选择打印机打印 | BarTender 出纸；状态已打印 | ☐ |
| B8 | 条码明细导出 CSV | 文件可打开，条数正确 | ☐ |
| B9 | 对已打印单补打 | 成功一次；再次补打被拒绝 | ☐ |
| B10 | 修改已生成过的规则 | 提示/重置 QA（按设计） | ☐ |
| B11 | QA 驳回 | 状态已驳回，可重新上传 | ☐ |

### 2.4 系统管理

| # | 步骤 | 预期结果 | 通过 |
|---|------|----------|------|
| S1 | 新建用户并分配角色 | 可登录且菜单正确 | ☐ |
| S2 | 管理员重置密码 | 用户下次须改密（若策略启用） | ☐ |
| S3 | 新建角色并分配权限 | 绑定用户后生效 | ☐ |

### 2.5 运维（IT）

| # | 步骤 | 预期结果 | 通过 |
|---|------|----------|------|
| O1 | 故意错误连接串 | 登录或启动失败，日志有记录 | ☐ |
| O2 | 缺少 Labels 模板打印 | 明确错误提示 | ☐ |
| O3 | 执行条码清理过程 | 超期数据按策略删除 | ☐ |
| O4 | 发布更高 version.json | 客户端提示更新（若启用） | ☐ |

## 4. 已知限制（初版）

- 会话失效后当前为**退出程序**，未实现回登录页
- 生产 / 物料四个菜单为占位
- 密码为 MD5 + 静态 Key
- 条码生成记录默认保留 30 天
