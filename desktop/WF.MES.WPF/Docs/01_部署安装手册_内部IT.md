# WF MES 部署安装手册（内部 IT）

> 版本：v1.0.1.2 · 读者：内部 IT · 平台：Windows 工位客户端 + SQL Server

---

## 1. 系统概述

WF MES 为 **.NET 8 WPF 桌面程序**（x86），通过 SQL Server 存业务数据。初版已实现：

- **条码管理**：客户、料号规则、资料审核、生成/打印、明细、补打
- **系统管理**：用户/角色/菜单在 **Web 后台**维护（桌面端不再内置 Admin）
- **生产 / 物料**：菜单已注册，页面为占位（待后续版本）

---

## 2. 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11 或 Windows Server（工位） |
| 运行时 | [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0) **x86** |
| 数据库 | SQL Server 2016+（库名 **MES**） |
| 后端 API | WF.MES.Api（工位可访问，供登录/菜单） |
| 打印工位 | **BarTender**（Automation/Enterprise，需 COM） |
| 网络 | 工位可访问 SQL Server 与 **API**；更新服务器（可选） |

> 程序编译目标为 **x86**，请安装 x86 版 Desktop Runtime，与 BarTender 32 位 COM 兼容。

## 3. 数据库与 API

### 3.1 全新安装

按 [database/README.md](../../../../database/README.md) 执行，推荐：

```text
database/sql/00_rebuild_all.sql
```

### 3.2 默认账号（与 Web/API 一致）

| 账号 | 默认密码 | 说明 |
|------|----------|------|
| `admin` | `Admin@123` | 全菜单 |
| `operator` | `Operator@123` | 生产+条码（按角色授权）；**首次登录须改密** |

密码由 API **PBKDF2** 校验，与 Web 相同。

## 4. 客户端部署

### 4.1 获取程序

- **开发构建**：`dotnet publish WF.MES.WPF/WF.MES.WPF.csproj -c Release`
- **发布包**：解压 `WF.MES.WPF.zip` 到固定目录，例如 `C:\WF.MES\`

目录内应包含：

- `WF.MES.WPF.exe`
- `WF.MES.Updater.exe`
- `appsettings.json`（需自行配置）
- `Labels\`（BarTender 模板，按料号放置 `{料号}.btw`）
- `update\version.json`（可选，本地更新清单）

### 4.2 配置文件

1. 复制 `appsettings.example.json` 为 `appsettings.json`
2. 至少配置：

| 配置项 | 说明 |
|--------|------|
| `Api:BaseUrl` | 后端 API 根地址（如 `http://服务器:5088` 或 `https://服务器:7143`） |
| `ConnectionStrings:WfMesDb` | MES 库连接串（条码 SqlSugar 直连） |
| `Session:HeartbeatIntervalSeconds` | 调用 `/auth/info` 间隔（秒） |
| `Update:CheckUrl` | 更新清单 URL 或本地 `update\version.json` 路径 |
| `Sop:PdfUrl` | SOP 文档路径（可选） |

示例见仓库 `appsettings.example.json`。

### 4.3 标签模板（BarTender）

- 路径：`{程序目录}\Labels\{料号}.btw`
- 模板内条码数据库字段名须为 **`Barcode`**（或与代码常量一致）
- 新料号上线前由工艺/条码维护人员提供 `.btw` 文件

### 4.4 首次启动

1. 双击 `WF.MES.WPF.exe`
2. 同一台电脑**不可重复打开**（单实例 Mutex）
3. 使用 `admin` 登录，确认侧栏菜单与数据库连通
4. 修改默认密码，在「角色权限分配」中为业务角色勾选菜单与受控按钮

## 5. 权限模型（IT 配置要点）

```
用户 → 角色 → 模组 → 菜单（页面能否进入）
              └→ 受控按钮（页面内敏感操作，Require_Grant=1）
```

| 类型 | 说明 | 示例 |
|------|------|------|
| 菜单权限 | 无权限则侧栏不显示 | 条码打印、资料审核 |
| 受控按钮 | 无权限则按钮不可用 / Service 拒绝 | 保存附件、审核、保存角色权限 |
| 常规按钮 | 有菜单即可用 | 客户保存、规则保存、刷新 |

`ADMIN` 角色自动拥有全部受控操作，无需逐条勾选。

当前受控操作码：

- `System.RolePermission.Save` — 保存角色权限
- `Barcode.QaReview.SaveAttachments` — 上传资料
- `Barcode.QaReview.Review` — QA 确认/驳回

## 6. 在线更新（可选）

1. 内网放置 `version.json` 与 `WF.MES.WPF.zip`
2. `appsettings.json` 配置 `Update:CheckUrl`
3. 客户端登录前检查版本；有新版本时下载并调用 `WF.MES.Updater.exe` 覆盖

`update/version.json` 示例：

```json
{
  "version": "1.0.1.2",
  "downloadUrl": "http://内网地址/updates/WF.MES.WPF.zip",
  "releaseNotes": "初版发布"
}
```

## 7. 会话与单实例

| 机制 | 作用 |
|------|------|
| **SingleInstanceGuard** | 同机仅允许一个客户端进程 |
| **Sys_UserSession** | 同账号仅一个活跃会话；异机登录需确认踢人 |
| **心跳** | 超时未心跳则会话失效，客户端提示后退出 |
