# WF MES 运维与故障排查（内部 IT）

> 版本：v1.0.1.2 · 读者：内部 IT

---

## 1. 日常运维项

| 周期 | 事项 | 说明 |
|------|------|------|
| 每日 | 检查 SQL Server 可用性 | 客户端登录、Shell 左下角/健康检查 |
| 每日 | 条码清理计划任务 | `sp_Barcode_PurgeExpired`，默认保留 30 天 |
| 按需 | 发布新版本 | 更新 `version.json` 与 zip 包 |
| 按需 | 备份 NMES 库 | 至少含条码与系统表 |
| 按需 | 新增料号模板 | `Labels\{料号}.btw` |
| 按需 | 角色权限调整 | 新人员入职 / 岗位变更 |

## 2. 日志

- 路径：`{程序目录}\Logs\log-YYYYMMDD.txt`
- 引擎：Serilog（级别与滚动见 `appsettings.json`）
- 启动、登录、条码生成、打印、未捕获异常均会写入

### 2.1 常用检索关键字

| 关键字 | 含义 |
|--------|------|
| `登录成功` / `登录失败` | 认证 |
| `会话已失效` | 心跳失败 |
| `BarTender` | 打印 COM |
| `SQL:` | Debug 级别 SQL（若开启） |
| `未捕获异常` | 客户端错误 |

---

## 3. 条码数据保留与清理

- 策略常量：30 天（`BarcodeRetentionPolicy.RetentionDays`）
- 存储过程：`dbo.sp_Barcode_PurgeExpired`（见 `Barcode_PurgeRetention.sql`）
- 计划任务示例：`Scripts/scheduled-tasks/`

### 3.1 注册计划任务（示例）

1. 复制 `purge_barcode.env.example.cmd` 为 `purge_barcode.env.cmd`，填写 SQL 连接
2. 以管理员运行 `register_purge_scheduled_task.ps1`
3. 确认任务每日执行成功（任务计划程序 / SQL 日志）

---

## 4. 在线更新发布

1. `dotnet publish` 或构建 Release 输出，打 zip（含 `WF.MES.WPF.exe`、`WF.MES.Updater.exe`、`Labels` 等）
2. 上传至内网静态目录
3. 更新 `version.json` 的 `version` 与 `downloadUrl`
4. 客户端下次启动或登录流程中会检测并提示更新

注意：更新包应保留各工位各自的 `appsettings.json`（不要被 zip 覆盖生产密码与连接串）。

## 5. 故障排查

### 5.1 无法启动 / 闪退

| 现象 | 排查 |
|------|------|
| 提示缺少 .NET | 安装 .NET 8 Desktop Runtime **x86** |
| 重复启动提示 | 正常；结束已有进程或任务管理器关闭 `WF.MES.WPF.exe` |
| 启动即退出 | 查看 `Logs\` 最新文件；检查 `appsettings.json` 是否存在且 JSON 合法 |

### 5.2 无法登录

| 现象 | 排查 |
|------|------|
| 账号密码错误 | 确认 API 可用；Web 后台重置密码；桌面与 Web 共用 PBKDF2 账号 |
| 数据库错误 | 测试连接串；SQL 服务是否启动；防火墙 1433 |
| 异机登录冲突 | 正常策略；确认踢人或等待超时 |

### 5.3 菜单空白 / 功能进不去

- 用户是否分配角色
- 角色是否勾选 **模组 + 菜单**（仅有模组无菜单时侧栏为空）
- 重新登录刷新权限

### 5.4 打印失败

| 现象 | 排查 |
|------|------|
| 打印机列表为空 | Windows 是否安装打印机驱动 |
| 模板不存在 | `Labels\{料号}.btw` 是否存在 |
| BarTender COM 错误 | 是否安装 BarTender；日志中 `BarTender.Application` |
| 字段绑定失败 | 模板主字段是否为 `Barcode` |

### 5.5 条码生成失败

| 提示 | 排查 |
|------|------|
| 待 QA 确认 | `Barcode_MaterialRule.Qa_Status` 须为已确认 |
| 流水号超出范围 | 规则段位数/进制不足，调整规则 |
| 条码重复 | 查 `Barcode_Record` 唯一约束；联系开发（极少见） |

### 5.6 会话频繁退出

- 调大 `Session:TimeoutMinutes`（需评估安全）
- 检查网络是否导致心跳 UPDATE 失败
- 是否有人在其他工位登录同一账号

## 6. 数据库备份建议

**建议每日全量或差异备份**，至少包含：

- 系统：`Sys_*`
- 条码：`Barcode_*`

恢复后验证：登录、抽查生成单与条码明细、权限树。

---

## 7. 安全注意

- 默认种子密码 `123456` 仅限首次安装，**上线前必须修改**
- `appsettings.json` 含数据库口令，勿提交 Git（已在 `.gitignore`）
- 密码由 Backend API（PBKDF2）统一校验，与 Web 一致；重置请在 Web 用户管理操作
- Service 层对部分操作有受控按钮校验，但仍建议 IT 正确分配最小权限
